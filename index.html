  6. Question: How do I quickly send you an encrypted email?
    • Answer: If you lack the time to read the GNU Privacy Guide, then please perform the following steps.
      1. Save one of the GPG keys listed above to a file called AlexSharif.pub.
      2. To import the key, run gpg --import AlexSharif.pub
      3. Compose a message and save it as an appropriate file with a distinct name (for example ) message.txt.
      4. Execute the command gpg --armor --encrypt --recipient alexander.a.sharif@gmail.com message.txt
      5. If you have performed the following correctly, then an ASCII armored file called message.txt.asc should appear in whatever working directory you ran the command listed above in. The encrypted file message.txt.asc is the cipher text that can be securely sent to me via email.
  7. Question: How do I send you an encrypted email so I too can get in on the fun?
    • Answer: If you lack the time to read the GNU Privacy Guide, then please perform the following steps.
      1. Run the following code to generate your public-private key pair: gpg --full-generate-key.
      2. Enter the required information such as your name and, most importantly the email that you plan to use (GPG operates primarily on email identifiers).
      3. Compose a message and save it as an appropriate file with a distinct name (for example ) message.txt.
      4. Execute the command gpg --armor --encrypt --recipient alexander.a.sharif@gmail.com message.txt
      5. If you have performed the following correctly, then an ASCII armored file called message.txt.asc should appear in whatever working directory you ran the command listed above in. Since the encrypted file message.txt.asc is the cipher text that can be securely sent to me via email and decrypted using my secret key, two way secure communications can now be established.
